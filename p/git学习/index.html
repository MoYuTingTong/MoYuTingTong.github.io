分区与工作流

Git分区

Git的分区包括工作区、暂存区、本地仓库（本地版本库）、远程仓库（远程版本库）。

  1. 工作区（Workspace）
    工作区即本地代码所在的目录，同时也是存放 .git/ （本地仓库）的目录。

  2. 暂存区（Index/Stage）
    暂存区是工作区和本地仓库的缓存空间，里面记录着即将提交给本地仓库（版本库）的文件修改信息，.git/ 目录里的index文件就是暂存区。

  3. 本地仓库（Repository）
    本地仓库也称本地库或版本库，存放了本地的所有版本（commit提交记录），本地仓库的文件都在 .git/ 目录中。

  4. 远程仓库（Remote）
    远程仓库在网络上，GitHub、Gitee和GitLab都能创建远程仓库，和本地仓库一样，远程仓库存放的也是不同的代码版本，只是这些版本可以来自多个本地仓库。

git工作流

暂存区文件

Git文件状态

在管理暂存区之前，我们还需要了解工作区文件的几种状态：

Untracked未跟踪，工作区中没有加入过暂存区的文件，不参与版本控制;
Unmodified未修改，加入版本控制，但和版本库中文件快照相同；
Modified已修改，加入版本控制，而且和上次加入版本库时的快照不同；
Staged已暂存，下一步可以提交到本地仓库（版本库）

git status命令可以用来查看工作区文件当前的状态：

1
2
3
4
5

git status [filename]    #查看指定文件的状态

git status    #查看所有文件状态

git status -s     #精简的方式显示文件状态

添加文件到暂存区 

1
2
3

git add [filename]    #添加指定文件到暂存区

git add -A    #添加工作区所有文件到暂存区

删除暂存区的文件 

1

git rm --cached [filename]    #将指定文件从暂存区移除

文件修改管理

Modified状态下对比文件：
git diff对比不同的状态下的同一文件
diff对比两个不同的文件

提交文件到本地仓库

提交命令 

1
2
3

git commit -m “message” [filename]    #添加指定文件到本地仓库

git commit -m “message”    #添加所有文件到本地仓库

查看日志 

1
2
3
4
5
6
7

git log     #查看提交记录

git log --oneline     #一行显示，只显示哈希值的前7位

git reflog     #显示历史提交版本与当前版本的间隔数

推送到远程仓库

创建远程仓库

添加远程仓库地址 

1

git remote add <name> <url>      #<name>为远程仓库的自定义本地别名

<url>为为仓库网络地址：
如果用https的方式上传代码，需要添加远程仓库https地址；
如果用ssh上传代码，则url填远程仓库ssh地址。

查看远程仓库地址 

1
2
3

git remote -v

git config -l

删除远程仓库地址 

1

git remote remove <name>       #删除本地别名为<name>的远程地址
